# General Development Rules

## Core Principles

### Understand Before Changing
- **ALWAYS** read relevant files before modifying
- **ASK** clarifying questions when uncertain
- **PRIME** the workspace before starting work

### Follow Existing Patterns
- **MATCH** existing code style and conventions
- **REUSE** existing patterns and abstractions
- **CONSISTENCY** over personal preference

### Minimal Changes
- **CHANGE** only what's necessary
- **FOCUS** on the task at hand
- **REFACTOR** only when explicitly requested

### Test as You Code
- **WRITE** tests alongside implementation
- **RUN** tests frequently
- **FIX** failures immediately

## Code Quality

- **Self-documenting:** Clear names > comments
- **Simple:** Prefer simple over clever
- **Maintainable:** Small functions, low coupling

## File Organization

- **FOLLOW** standard project structure
- **USE** descriptive file names
- **GROUP** related files together

## Error Handling

- **HANDLE** errors gracefully
- **PROVIDE** meaningful error messages
- **LOG** errors with context
- **FAIL** safely (secure by default)
